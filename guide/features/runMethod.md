# Run Method

Run Method 支持直接运行 Java/Kotlin 方法，根据方法签名生成参数，并调用运行中的应用内的 Spring/Solon Bean 方法。

::: tip 前提

目标应用需要处于运行状态，目标方法需要是 `public`、非 `static`、非构造方法，并且所在类需要能在运行中的应用容器中解析为 Bean。

:::

## 触发方法

![](/img/2026.1.1/runMethod.png)

在 Java/Kotlin 方法左侧点击  图标，或在方法上触发 `RunMethod(FR)`，插件会打开 `RunMethod` Tab，并根据方法参数生成对应的参数编辑器。

## 参数和响应

每个方法参数会生成一个独立的参数编辑 Tab，参数内容使用 JSON 编辑器展示。修改参数后点击 `Run`，插件会调用运行中的应用内 Bean 方法，并在 `Response` 中展示返回结果。

如果返回结果是 JSON，会使用 JSON 编辑器展示；非 JSON 结果会使用文本编辑器展示。

## 保存方法

Run Method 工具栏支持保存当前方法和参数数据。保存后，同一个方法可以继续保存不同参数组，用于维护多组调用参数。

支持的操作包括：

* 定位到方法源码
* 重新生成参数
* 清空当前方法、参数和响应
* 保存方法和当前参数
* 保存当前参数为参数组
* 切换已保存的参数组

## Methods

![](/img/2026.1.1/methods.png)

`Methods` Tab 用于查看和管理已保存的方法。已保存的方法按类分组展示，方法行提供定位源码和运行操作。

双击已保存的方法会打开对应方法并定位到源码。右键方法可以打开参数分组菜单，选择已保存的参数组后会切换到对应参数数据。
